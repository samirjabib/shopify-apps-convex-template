# Convex Adapter for Shopify React Router Boilerplate

**Status:** Draft — design approved 2026-04-22
**Author:** samir
**Scope:** Replace Prisma + SQLite with Convex Cloud as backend for session storage and app data.

## 1. Goal

Adapt the existing Shopify React Router v7 boilerplate (currently using Prisma + SQLite for Shopify session storage) to use Convex Cloud as the backend for:

- Shopify session persistence (replaces `@shopify/shopify-app-session-storage-prisma`)
- App business data (starter `shops` table)
- Reactive client queries via `ConvexReactClient` / `useQuery`

Prisma is fully removed. No dual system.

## 2. Non-Goals

- Migrating existing session data from `dev.sqlite` (boilerplate starter — no live data)
- Automated test suite (not present in upstream, not adding)
- Convex self-hosted deployment (Cloud only)
- Product sync, webhooks beyond existing two (`app/uninstalled`, `app/scopes_update`)

## 3. Architecture

```
┌─────────────────────────────────────────────────┐
│  React Router v7 (SSR, server + client)         │
│                                                 │
│  ┌─────────────┐      ┌──────────────────────┐  │
│  │ Loaders /   │─────▶│ ConvexHttpClient     │──┼──▶ Convex Cloud
│  │ Actions /   │      │ (server, admin auth) │  │   ┌──────────────┐
│  │ Webhooks    │      │                      │  │   │ sessions     │
│  └─────────────┘      └──────────────────────┘  │   │ shops        │
│                                                 │   │ functions/   │
│  ┌─────────────┐      ┌──────────────────────┐  │   │  - internal  │
│  │ Components  │─────▶│ ConvexReactClient    │──┼──▶│  - public    │
│  │ useQuery()  │      │ (browser, JWT auth)  │  │   └──────────────┘
│  └─────────────┘      └──────────────────────┘  │
└─────────────────────────────────────────────────┘
```

- Server path: access-token-bearing functions (session storage) — uses Convex admin key
- Client path: reactive queries — each call validates Shopify session token JWT

```ts
export default defineSchema({

    refreshTokenExpires: v.optional(v.number()),

    .index("by_shop", ["shop"]),

  shops: defineTable({

    installedAt: v.number(),
```

| Prisma | Convex | Notes |
|--------|--------|-------|

## 6. Security Model

### Function tiers

**Internal functions** (`internalMutation`, `internalQuery`) — not exposed over HTTP.

- All session storage (access tokens never reachable by client)
- `sessions.storeInternal`
- `sessions.loadBySessionIdInternal`
- `sessions.deleteBySessionIdInternal`
- `sessions.deleteManyInternal`
- `sessions.findByShopInternal`
- `sessions.deleteByShopInternal` (webhook cleanup)
- `sessions.updateScopeInternal` (webhook scope update)
- `shops.upsertInternal` (called during auth)

Called only from `ConvexHttpClient` with admin auth from React Router server.

**Public functions** (`query`, `mutation`) — exposed to browser.

- All take `sessionToken: v.string()` as first arg
- First line: `const { shop } = await requireShopifyAuth(token)`
- Shop extracted from verified JWT `dest`, never from client-supplied arg
- Starter: `shops.get` (returns current shop's metadata)

### `requireShopifyAuth` implementation

`convex/lib/auth.ts`:

```ts
import { ConvexError } from "convex/values";

function base64UrlDecode(s: string): Uint8Array {
  const pad = s.length % 4;
  const b64 = (s + "===".slice(0, pad ? 4 - pad : 0)).replace(/-/g, "+").replace(/_/g, "/");
  const bin = atob(b64);
  const out = new Uint8Array(bin.length);
  for (let i = 0; i < bin.length; i++) out[i] = bin.charCodeAt(i);
  return out;
}

function timingSafeEqual(a: Uint8Array, b: Uint8Array): boolean {
  if (a.length !== b.length) return false;
  let diff = 0;
  for (let i = 0; i < a.length; i++) diff |= a[i] ^ b[i];
  return diff === 0;
}

export async function requireShopifyAuth(token: string): Promise<{ shop: string; userId?: string }> {
  const secret = process.env.SHOPIFY_API_SECRET;
  const apiKey = process.env.SHOPIFY_API_KEY;
  if (!secret || !apiKey) throw new ConvexError("Server misconfigured");

  const parts = token.split(".");
  if (parts.length !== 3) throw new ConvexError("Malformed token");
  const [headerB64, payloadB64, sigB64] = parts;

  const key = await crypto.subtle.importKey(
    "raw",
    new TextEncoder().encode(secret),
    { name: "HMAC", hash: "SHA-256" },
    false,
    ["sign"],
  );
  const signed = await crypto.subtle.sign(
    "HMAC",
    key,
    new TextEncoder().encode(`${headerB64}.${payloadB64}`),
  );
  if (!timingSafeEqual(new Uint8Array(signed), base64UrlDecode(sigB64))) {
    throw new ConvexError("Invalid token signature");
  }

  const payload = JSON.parse(new TextDecoder().decode(base64UrlDecode(payloadB64)));
  const now = Math.floor(Date.now() / 1000);
  if (payload.exp < now) throw new ConvexError("Token expired");
  if (payload.nbf > now) throw new ConvexError("Token not yet valid");
  if (payload.aud !== apiKey) throw new ConvexError("Wrong audience");
  if (!payload.dest) throw new ConvexError("Missing dest");

  const shop = new URL(payload.dest).hostname;
  return { shop, userId: payload.sub };
}
```

### Convex environment variables

Set via `npx convex env set`:

- `SHOPIFY_API_SECRET` — for JWT HMAC verification
- `SHOPIFY_API_KEY` — for JWT audience check

### Server → internal function auth

`ConvexHttpClient` must authenticate as admin to call internal functions. Uses `CONVEX_DEPLOY_KEY` (production deploy key, or preview deploy key for dev). Set in React Router server env only.

`app/convex.server.ts`:

```ts
import { ConvexHttpClient } from "convex/browser";

const client = new ConvexHttpClient(process.env.CONVEX_URL!);
client.setAdminAuth(process.env.CONVEX_DEPLOY_KEY!);

export default client;
```

**Verify during implementation:** confirm `ConvexHttpClient.setAdminAuth()` grants internal-function access. If API differs, fall back to exposing a single HTTP action gated by shared-secret header.

### Client session token handling

App Bridge tokens expire in ~60 seconds. Wrap token retrieval in a hook:

```ts
// app/lib/session-token.client.ts
import { useEffect, useState } from "react";
import { useAppBridge } from "@shopify/app-bridge-react";

export function useShopifySessionToken(): string | null {
  const app = useAppBridge();
  const [token, setToken] = useState<string | null>(null);

  useEffect(() => {
    let cancelled = false;
    const refresh = async () => {
      const t = await app.idToken();
      if (!cancelled) setToken(t);
    };
    refresh();
    const id = setInterval(refresh, 30_000); // refresh every 30s
    return () => { cancelled = true; clearInterval(id); };
  }, [app]);

  return token;
}
```

Consumer:

```tsx
const token = useShopifySessionToken();
const shop = useQuery(api.shops.get, token ? { sessionToken: token } : "skip");
```

### Threat coverage

| Threat | Mitigation |
|--------|-----------|
| Attacker with Convex URL calls session functions | Session functions are `internal`, require admin auth |
| Attacker with Convex URL reads another shop's data via public function | Public functions reject calls without valid Shopify JWT; shop extracted from JWT, not client arg |
| Replay of expired session token | `exp` / `nbf` check in `requireShopifyAuth` |
| Cross-app token replay | `aud` check against `SHOPIFY_API_KEY` |
| Secret leak via client bundle | `SHOPIFY_API_SECRET` only in Convex env; `CONVEX_DEPLOY_KEY` only in server env |

## 7. Session Storage Adapter

`app/lib/session-storage.server.ts`:

```ts
import type { SessionStorage } from "@shopify/shopify-app-session-storage";
import { Session } from "@shopify/shopify-api";
import type { ConvexHttpClient } from "convex/browser";
import { internal } from "../../convex/_generated/api";

export class ConvexSessionStorage implements SessionStorage {
  constructor(private client: ConvexHttpClient) {}

  async storeSession(session: Session): Promise<boolean> {
    await this.client.mutation(internal.sessions.storeInternal, {
      session: serialize(session),
    });
    return true;
  }

  async loadSession(id: string): Promise<Session | undefined> {
    const row = await this.client.query(internal.sessions.loadBySessionIdInternal, {
      sessionId: id,
    });
    return row ? deserialize(row) : undefined;
  }

  async deleteSession(id: string): Promise<boolean> {
    await this.client.mutation(internal.sessions.deleteBySessionIdInternal, {
      sessionId: id,
    });
    return true;
  }

  async deleteSessions(ids: string[]): Promise<boolean> {
    await this.client.mutation(internal.sessions.deleteManyInternal, { sessionIds: ids });
    return true;
  }

  async findSessionsByShop(shop: string): Promise<Session[]> {
    const rows = await this.client.query(internal.sessions.findByShopInternal, { shop });
    return rows.map(deserialize);
  }
}

function serialize(s: Session) {
  return {
    sessionId: s.id,
    shop: s.shop,
    state: s.state,
    isOnline: s.isOnline,
    scope: s.scope ?? undefined,
    expires: s.expires ? s.expires.getTime() : undefined,
    accessToken: s.accessToken ?? "",
    userId: s.onlineAccessInfo?.associated_user?.id
      ? String(s.onlineAccessInfo.associated_user.id)
      : undefined,
    firstName: s.onlineAccessInfo?.associated_user?.first_name ?? undefined,
    lastName: s.onlineAccessInfo?.associated_user?.last_name ?? undefined,
    email: s.onlineAccessInfo?.associated_user?.email ?? undefined,
    accountOwner: s.onlineAccessInfo?.associated_user?.account_owner ?? false,
    locale: s.onlineAccessInfo?.associated_user?.locale ?? undefined,
    collaborator: s.onlineAccessInfo?.associated_user?.collaborator ?? undefined,
    emailVerified: s.onlineAccessInfo?.associated_user?.email_verified ?? undefined,
  };
}

function deserialize(row: any): Session {
  const s = new Session({
    id: row.sessionId,
    shop: row.shop,
    state: row.state,
    isOnline: row.isOnline,
  });
  s.scope = row.scope;
  s.expires = row.expires ? new Date(row.expires) : undefined;
  s.accessToken = row.accessToken;
  if (row.userId) {
    s.onlineAccessInfo = {
      associated_user: {
        id: Number(row.userId),
        first_name: row.firstName,
        last_name: row.lastName,
        email: row.email,
        account_owner: row.accountOwner,
        locale: row.locale,
        collaborator: row.collaborator,
        email_verified: row.emailVerified,
      },
    } as any;
  }
  return s;
}
```

Serialize/deserialize handle `BigInt ↔ string` and `Date ↔ ms epoch` conversions.

## 8. Dev Workflow

### First-time setup

1. `npm install` (includes `convex`)
2. `npx convex dev` — prompts login, creates project, writes `CONVEX_URL` to `.env.local`
3. `npx convex env set SHOPIFY_API_SECRET <value>`
4. `npx convex env set SHOPIFY_API_KEY <value>`
5. Generate deploy key: Convex dashboard → Settings → Deploy Keys → create preview key
6. Add to `.env`: `CONVEX_DEPLOY_KEY=<key>`, `VITE_CONVEX_URL=<same as CONVEX_URL>`
7. Two terminals:
   - `npm run dev` (Shopify)
   - `npm run convex:dev` (Convex codegen + deploy functions)

```ts
      scope: current.toString(),
    });

  return new Response();

```

## 10. Error Handling

- Convex client errors in `ConvexSessionStorage` methods → log + re-throw (Shopify lib's expectation)
- Webhook handlers wrap mutations in `try/catch`, log errors, always return `200` (avoid Shopify retry storms for transient Convex blips)
- Client `useQuery` errors: let Convex/React Router error boundary handle — no special UI in boilerplate

## 11. Testing

- No automated test suite (not present in boilerplate, out of scope)
- Manual verification checklist:
  1. Fresh install: `rm -rf node_modules && npm install && npx convex dev` creates deployment
  2. Shopify OAuth: install app on dev store → session row appears in Convex dashboard
  3. Session persistence: restart `npm run dev` → app still authenticated (session read from Convex, not memory)
  4. Scope update: trigger scope change → `scopes_update` webhook fires → Convex session row shows updated scope
  5. Uninstall: uninstall app → `app/uninstalled` webhook fires → Convex session rows for that shop deleted
  6. Public function auth: call `api.shops.get` from browser dev console without token → rejected with `ConvexError("Malformed token")`
  7. Public function auth: call with expired token → rejected with `ConvexError("Token expired")`

## 12. Open Verification Items

These need confirmation during implementation (not blockers for design approval):

1. **`ConvexHttpClient.setAdminAuth()` API** — verify current method name and that it grants internal-function access. If API has changed, fall back: expose session storage as an HTTP action (`httpAction`) gated by shared secret header, called via `fetch` from server.
2. **Convex runtime Web Crypto support** — confirm `crypto.subtle.importKey` + HMAC-SHA256 work in Convex V8 isolate. Docs indicate yes; verify on first `requireShopifyAuth` call.
3. **Session token `aud` claim** — confirm Shopify App Bridge session tokens set `aud = API_KEY`. If instead `iss`, adjust check.
4. **`onlineAccessInfo` shape in `Session` class** — verify field names match serializer. Prisma schema maps per-field; if Shopify library uses a getter, adjust.

## 13. Risks

| Risk | Mitigation |
|------|-----------|
| Convex Cloud outage blocks OAuth + every request | Accept — same risk profile as any managed backend; document in README |
| Token refresh race (expired mid-request) | Convex returns `Token expired` error → client re-fetches token → retry. `useShopifySessionToken` refresh interval (30s) shorter than token lifetime (60s) |
| Admin key leak via server logs | Never log `CONVEX_DEPLOY_KEY`; use `.env.local` (gitignored); rotate via Convex dashboard |
| Breaking change in Convex SDK between versions | Pin `convex` version in `package.json`, document update procedure |
